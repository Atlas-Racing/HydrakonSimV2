---
title: 3. ROS 2 Topics
---

This page lists all the active ROS 2 topics in the Hydrakon simulation environment, detailing their message types and function.

## Vehicle Control & State

| Topic Name | Message Type | Publisher | Description |
| :--- | :--- | :--- | :--- |
| `/cmd_vel` | `geometry_msgs/msg/Twist` | `manual_control_node` | Represents the **user's control intent**. Linear X corresponds to throttle/brake, Angular Z corresponds to steering. |
| `/odom` | `nav_msgs/msg/Odometry` | `carla_to_ros_bridge` | The vehicle's **ground truth state**. Position comes from CARLA's physics engine; Orientation and Angular Velocity come from the simulated IMU. |
| `/imu` | `sensor_msgs/msg/Imu` | `ins_node` | Raw data from the simulated Inertial Measurement Unit (Accelerometer + Gyroscope) with simulated noise. |
| `/joint_states` | `sensor_msgs/msg/JointState` | `carla_to_ros_bridge` | Contains the calculated angles for the vehicle's wheels (steering and rotation) to animate the URDF model in RViz. |

## Sensors (Perception)

| Topic Name | Message Type | Publisher | Description |
| :--- | :--- | :--- | :--- |
| `/camera/raw` | `sensor_msgs/msg/Image` | `carla_rgb_camera_spawner` | Raw RGB video stream from the front-facing camera. |
| `/camera/depth` | `sensor_msgs/msg/Image` | `carla_camera_spawner` | Depth map stream, where pixel intensity corresponds to distance in meters. |
| `/carla/lidar` | `sensor_msgs/msg/PointCloud2` | `lidar_node` | 3D Point Cloud generated by the simulated LiDAR sensor (64-channel ray cast). |

## Perception Outputs

| Topic Name | Message Type | Publisher | Description |
| :--- | :--- | :--- | :--- |
| `/camera/cone_detections` | `vision_msgs/msg/Detection2DArray` | `cone_detector` | List of 2D bounding boxes for cones detected in the RGB image by the YOLO model. |
| `/camera/cone_detections_image`| `sensor_msgs/msg/Image` | `cone_detector` | Debug image showing the RGB feed with bounding boxes and labels drawn on top. |
| `/camera/cone_markers` | `visualization_msgs/msg/MarkerArray` | `cone_locator` | 3D visual markers (cylinders) representing estimated cone positions derived from Camera + Depth. |

## Infrastructure & Visualization

| Topic Name | Message Type | Publisher | Description |
| :--- | :--- | :--- | :--- |
| `/tf` | `tf2_msgs/msg/TFMessage` | `robot_state_publisher`, `carla_to_ros_bridge` | Dynamic transforms (e.g., `odom` -> `base_footprint`, `base_link` -> Wheels). |
| `/tf_static` | `tf2_msgs/msg/TFMessage` | `static_transform_publisher` | Fixed transforms (e.g., `base_link` -> `lidar_link`, `base_link` -> `sensor_frames`). |
| `/robot_description` | `std_msgs/msg/String` | `robot_state_publisher` | The XML content of the URDF robot model. |
| `/parameter_events` | `rcl_interfaces/msg/ParameterEvent` | (All Nodes) | Updates when a ROS 2 parameter is changed on any node. |
| `/rosout` | `rcl_interfaces/msg/Log` | (All Nodes) | System-wide logging output. |
| `/diagnostics` | `diagnostic_msgs/msg/DiagnosticArray` | `greenwave_monitor` | System health and diagnostic information for monitored topics. |
| `/clicked_point` | `geometry_msgs/msg/PointStamped` | `rviz2` | Published when the user uses the "Publish Point" tool in RViz. |
| `/goal_pose` | `geometry_msgs/msg/PoseStamped` | `rviz2` | Published when the user sets a "2D Nav Goal" in RViz. |
| `/initialpose` | `geometry_msgs/msg/PoseWithCovarianceStamped` | `rviz2` | Published when the user sets a "2D Pose Estimate" in RViz. |
